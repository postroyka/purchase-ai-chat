## 1. Для сотрудника — как загрузить счёт

### Вход
- **Внутри Битрикс24** (приложение в левом меню портала) — вход не нужен, сессия открывается сама.
- **По отдельной ссылке** (вне портала) — форма **«Вход»**: «Логин» и «Пароль» (выдаёт администратор).

### Загрузка
1. Перетащите файлы в зону **«Перетащите файлы сюда»** или нажмите, чтобы выбрать. Можно несколько — **до 10 файлов, по 20 МБ**.
2. Форматы: **PDF, фото (JPG/PNG), Excel (XLSX/XLS), Word (DOCX)**.
3. Нажмите **«Загрузить N файлов»**. _(До нажатия можно убрать лишний файл крестиком рядом с ним.)_
4. Идёт обработка — виден прогресс и таймер ⏱ (плюс общее время импорта). **Разбор одного файла может занять до нескольких минут — это нормально** (PDF/фото распознаются через OCR). Жёсткий предел на файл — **6 минут**: если не уложился, разбор останавливается с ошибкой. Файлы обрабатываются по очереди; импорт можно прервать кнопкой **«Остановить импорт»**. Вкладку лучше не закрывать (приложение предупредит).

### Результат — по каждому файлу
- ✅ **«Сделка #N»** + кнопка **«Открыть сделку»** — готово, сделка создана в воронке «Закупки» _(внутри Битрикс24 откроется карточкой-слайдером поверх портала; по внешней ссылке — в новой вкладке)_.
- ⚠️ **«Без сделки»** + причина — например «Валюта не BYN», «Иностранный поставщик», «Поставщик не найден», «Договор не найден». Это **не ошибка приложения**, а бизнес-правило (см. ниже).
- ❌ **Ошибка** — файл не удалось обработать (битый/нечитаемый/не тот тип). **Повторите загрузку; если повторяется — сообщите администратору.**
- **«Лог обработки»** (раскрыть) — что распознал агент и почему результат такой.

### Отзыв
Под каждым файлом: **👍 Хорошо** / **👎 Проблема** (+ комментарий — необязательно). Отзывы помогают улучшать разбор — пользуйтесь, особенно при 👎.

### Что важно знать (правила — кратко)
- Сделка создаётся в воронке **«Закупки»**, валюта **BYN**, НДС **20%**, цена из документа — **без НДС за 1 шт**, количество как в счёте, единица — **штуки** (намеренно).
- **Ответственный** за сделку — **вы** (тот, кто загрузил файл внутри Битрикс24). По внешней ссылке (вне портала) ответственный — пользователь по умолчанию (`PUBLIC_PAGE_RESPONSIBLE_USER_ID`).
- Поставщик ищется по **УНП** (реквизит в CRM). Нет такого УНП в CRM → «Поставщик не найден».
- Позиция добавляется, только если её **артикул есть в каталоге**. Нет → позиция не попадёт в сделку (видно в «Логе обработки»).
- Счёт **не в BYN** → сделка не создаётся («Валюта не BYN»).
- **Дубли не проверяются** — повторная загрузка того же файла создаёт новую сделку.

## 2. Запуск (go-live) — для владельца/админа

### Чек-лист перед запуском
- [ ] Прод поднят, `GET /health` → `{ "ok": true, "redis": "ok", "feedbackOutboxPending": 0 }`.
- [ ] Логины для входа **вне портала** заданы — `PUBLIC_PAGE_BASIC_AUTH_USER` / `PUBLIC_PAGE_BASIC_AUTH_PASS` в env (внутри Битрикс24 вход не нужен — сессия сама).
- [ ] Приёмочный прогон по [docs/MANUAL_TEST_PLAN.md](MANUAL_TEST_PLAN.md) (§0–§7).
- [ ] Репозиторий отзывов `GITHUB_FEEDBACK_REPO` — **приватный** (в issue попадает контекст заданий, #190).
- [ ] Экономика подтверждена с заказчиком (см. ниже).
- [ ] **Очистить метрики** после тестовых прогонов — `make metrics-reset` (удаляет только `metrics:*`), чтобы старт считался «с чистого листа».

### Подтвердить экономику (#75)
Дашборд `/metrics` показывает плитку **«Сэкономлено (нетто)»**, но по умолчанию ставка и время — **плейсхолдеры** (в предупреждении под плиткой прямо написано «не подтверждена заказчиком»). Уточнить у заказчика и проставить в env:
- `HOURLY_RATE_BYN` — сколько стоит компании **час работы менеджера** (оклад + налоги), BYN. _(сейчас по умолчанию `18`.)_
- `MINUTES_PER_POSITION` — сколько минут уходит на **ручной ввод одной позиции** счёта. _(сейчас по умолчанию `2`.)_

Курс USD→BYN берётся живой с НБ РБ (фолбэк — `USD_BYN_RATE`).

## 3. Как растить точность (по `/metrics`)

Дашборд **`/metrics`** — главный инструмент после запуска. Смотрите регулярно. Большинство улучшений точности — это **данные в CRM** (поставщики по УНП, договоры, каталог артикулов), а не код.

| Сигнал на дашборде | Что значит | Что сделать |
|---|---|---|
| Низкий **% успеха** | мало сделок относительно загруженных файлов | смотреть причины ниже + «Лог обработки» у файлов |
| **«Поставщики без матча (по УНП)»** | счёт от поставщика, которого нет в CRM по УНП | завести контрагента в CRM с верным УНП (реквизит `RQ_INN`) |
| **«Артикулы без матча»** | артикул из счёта не найден в каталоге | добавить артикул в каталог (или сверить написание) |
| **«Мультиматчи (взяли один из нескольких)»** | было >1 совпадение, взяли первый по id | проверить дубли поставщика/договора/товара в CRM |
| **«Иностранный поставщик»**, **«Валюта не BYN»** | бизнес-правило (намеренно отсекаем) | не дефект; если нужно иначе — согласовать правило |
| **«Договор не найден»** | у поставщика нет активного договора нужного типа | завести/проверить договор в CRM |
| **👎 от сотрудников** (Обратная связь) | конкретные жалобы на результат | разобрать по «Логу обработки» → поправить данные CRM или, осторожно, промпт |
| **«Скорость разбора»** — много «медленно» | разбор долгий | разбор сигналов — [docs/PARSING_PERFORMANCE.md](PARSING_PERFORMANCE.md) (#222) |

Промпт агента — `prompts/main.md` (менять осторожно и по согласованию; бизнес-правила в нём фиксированы).
