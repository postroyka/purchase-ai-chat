# Eval-набор агента (#93)

Замеряет **точность извлечения агента** на реальных счетах — то, чего не покрывают
unit-тесты (они проверяют код пайплайна, а не то, что модель правильно *прочитала*
документ). Закрывает риск №1 из оценки проекта: считает ли агент НДС по `÷1.2`,
матчит ли поставщика по УНП, отсекает ли не-BYN.

## Что где

| Файл | Роль | Где проверяется |
|---|---|---|
| `score.js` | чистая сверка результата агента с эталоном + проверка направления НДС | `backend/tests/eval-score.test.js` (в обычном `pnpm test`) |
| `run.js` | прогон реального пайплайна по фикстурам → скоринг → отчёт | только на сервере (нужна модель) |
| `scripts/samples/<name>.pdf` + `<name>.expected.json` | фикстуры + эталоны | — |

## Запуск

```bash
make eval                      # = node backend/eval/run.js
```

⚠️ **Только там, где поднят агент**: нужны `claude` CLI, ключ модели
(`ANTHROPIC_*` / провайдер), доступный MCP-сервер (`MCP_SERVER_URL`,
`NUXT_MCP_AUTH_TOKEN`) и извлечение текста (`pdftotext`/OCR). В обычном CI не гоняем —
каждая фикстура это платный вызов модели. Логика скоринга при этом покрыта unit-тестами
и в CI бежит.

Переменные: `EVAL_SAMPLES_DIR`, `EVAL_RESPONSIBLE_ID`, `AGENT_TIMEOUT_MS`.

## Формат `<name>.expected.json`

Отказ (валюта/поставщик/договор не прошли):
```json
{ "fixture": "etalon-invoice.pdf", "expect": "error", "error": "unsupported_currency", "notes": "почему" }
```

Успешная сделка:
```json
{
  "fixture": "byn-ok.pdf",
  "expect": "deal",
  "currency": "BYN",
  "supplier": { "unp": "191234567" },
  "items": [
    { "name": "Цемент М500", "priceExclVat": 100.0, "quantity": 10, "priceInclVatHint": 120.0 }
  ]
}
```

- `priceInclVatHint` (опц.) — цена С НДС из документа. Скорер проверит, что агент
  поделил на 1.2, а не умножил на 0.8 (баг #58). Если в документе цена и так без НДС —
  не задавать.
- Позиции сверяются позиционно (в порядке документа); имя — нестрого.

## Текущие фикстуры

- **`etalon-invoice.pdf`** → `unsupported_currency`. Это счёт **российского** поставщика
  (ООО «ЭКОДИЗАЙН», ИНН/КПП, ПАО Сбербанк Москва) в **RUB**. Корректное поведение —
  ОТКАЗ (правило 1: только BYN). Ловит подвох: «руб.» не различает RUB/BYN — сигнал в
  стране поставщика/банка.
