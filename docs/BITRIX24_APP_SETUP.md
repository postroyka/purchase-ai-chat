# Регистрация приложения в Битрикс24

Как подключить **procure-ai** к порталу Битрикс24 как локальное приложение. Эту часть
нельзя сделать кодом — она выполняется один раз вручную администратором портала в
интерфейсе Битрикс24.

> Термины ниже — это поля из карточки локального приложения в Битрикс24.

## Предусловия

- У вас есть развёрнутый UI-сервис приложения, доступный по HTTPS (далее `<APP_URL>`,
  например `https://procure.example.com`).
- Вы — администратор портала Битрикс24.

## Шаги

1. Откройте портал → раздел **«Разработчикам»** → **«Другое»** → **«Локальное приложение»**.
   Для локального приложения «добавить» и «установить» — это одно и то же действие.

2. Заполните пути обработчиков:

   | Поле | Значение | Зачем |
   |------|----------|-------|
   | **Путь вашего обработчика** | `<APP_URL>/` | Основной интерфейс. Битрикс24 встроит его в **левое меню** (раздел «Приложения»). |
   | **Путь для первоначальной установки** | `<APP_URL>/install` | Установщик. Битрикс24 показывает его при первом открытии; страница вызывает `installFinish()` и завершает установку. |

3. **«Использует только API»** — **выключено**. У приложения есть пользовательский
   интерфейс, поэтому установку подтверждает страница `/install`, а не callback.

4. Выдайте права (scope), которые нужны приложению:

   - `crm` — работа со сделками;
   - `user_brief` — профиль пользователя;
   - `tasks`;
   - `entity`.

5. Сохраните приложение.

## Что происходит после сохранения

- При первом открытии приложения администратором Битрикс24 показывает во фрейме
  страницу `/install` («Идёт установка…»). Она дожидается готовности фрейма и вызывает
  `installFinish()`.
- После этого Битрикс24 считает приложение установленным, перезагружает фрейм и открывает
  основной интерфейс (`/`). Приложение появляется в левом меню для всех сотрудников.

> ⚠️ Пока со страницы `/install` не вызван `installFinish()`, Битрикс24 считает приложение
> «не установленным»: интерфейс заблокирован, а обычные сотрудники видят сообщение
> «обратитесь к администратору, он не завершил установку».

## Чего установка пока НЕ делает

Страница `/install` сейчас только подтверждает установку. Донастройку портала (создание
полей сделки, регистрацию событий и виджетов) она **не выполняет** — это добавим позже,
когда будем расширять приложение.

Поля сделки при необходимости можно создать отдельным REST-методом
`shef:purchase.api.procureinstall.ensureSchema` (см. [b24-controller](../b24-controller/README.md)),
но в процесс установки он намеренно не подключён.

## Проверка

Статус установки можно проверить REST-методом `app.info` — поле `INSTALLED` должно быть
`true`.
