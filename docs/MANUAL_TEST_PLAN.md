# План ручного тестирования — procure-ai

Чек-лист приёмочной и портал-проверки: то, что **нельзя** покрыть автотестами и нужно прогнать
руками на тестовом портале Битрикс24 перед тиражом. Автотесты (`ui` vitest, `backend` vitest,
`b24-controller` PHPUnit, MCP-overlay) проверяют логику в изоляции; здесь — поведение целиком,
от загрузки файла до сделки в реальном портале.

Формат: **сценарий → шаги → ожидаемый результат**. Колонку «✓/✗» заполняйте при прогоне.

> Связанные доки: установка — [`BITRIX24_APP_SETUP.md`](BITRIX24_APP_SETUP.md); сервер —
> [`SERVER_SETUP.md`](SERVER_SETUP.md); сопровождение и откат — [`OPERATIONS.md`](OPERATIONS.md);
> обратная связь — [`FEEDBACK.md`](FEEDBACK.md); бот — [`B24_BOT.md`](B24_BOT.md); бизнес-правила —
> [`PROJECT_BRIEF.md`](PROJECT_BRIEF.md).

| # | Сценарий | Шаги | Ожидаемо | ✓/✗ |
|---|---|---|---|---|
| 4.1 | **Все форматы** | Прогнать по одному файлу: PDF, XLSX, DOCX, XLS, JPG, PNG. | Текст извлечён (office → python-хелпер, PDF/скан → OCR), сделка создана. |  |

| 4.4 | **Точность УНП (негатив)** | Счёт с УНП, которого нет в CRM (или отличается на одну цифру). | Поставщик **НЕ найден** (точное `=`, без ложного матча на похожий УНП). |  |

| 4.6 | **Позиции/НДС** | Счёт с известными артикулами. | Позиции привязаны: цена БЕЗ НДС (нетто) × количество, налог 20 %, флаг «НДС включён» = N (нетто, НДС сверху, #325) — цена 1-в-1 со счётом, единицы → штуки. _(Намеренно, бизнес-правило — не дефект.)_ |  |
| 4.7 | **Мультиматч (#195)** | Документ, где артикул/поставщик даёт >1 совпадения. | Берётся min-id; на `/metrics` (блок «Проблемы матчинга») растёт плитка «Мультиматчи». |  |
| 4.8 | **Артикул без матча (#195/#151)** | Позиция с артикулом не из каталога. | Позиция НЕ попадает в сделку; причина в отчёте; на `/metrics` — плитка «Артикулы без матча» (блок «Проблемы матчинга»). |  |
| 4.9 | **Не-BYN** | Счёт в USD/EUR. | Сделка НЕ создаётся; на странице результата бейдж «Без сделки» + причина (#192). |  |

| 4.15 | **Лимит конкуренции** | Превысить `MAX_CONCURRENT_JOBS` (по умолч. 2) одновременными загрузками. | Сверх лимита — `429` с телом **«Server busy — too many jobs in progress…»** (это НЕ rate-limit из §7.3). |  |
| 4.16 | **Убрать файл до загрузки (#238)** | Выбрать несколько файлов; у одного нажать крестик; затем нажать «Загрузить N файлов». | Выбор файлов **не** стартует загрузку сам; лишний файл убирается до отправки; уходят только оставшиеся. Доп.: кнопка «Загрузить» отсутствует, пока ничего не выбрано, и скрыта во время загрузки/обработки; метка склоняется («1 файл» / «3 файла» / «7 файлов»); удаление **последнего** файла прячет кнопку; после завершения задания кнопка НЕ всплывает повторно (выбор очищен). |  |

| 5.5 | **Приватность репо (#190)** | Временно сделать репо обратной связи публичным, перезапустить бэкенд. **→ После проверки вернуть в private, снова перезапустить и убедиться, что WARNING исчез.** | В логах старта при публичном репо — WARNING «feedback repo … is PUBLIC»; после возврата в private WARNING пропадает. |  |
| 5.6 | **Канал не настроен** | Снять `GITHUB_FEEDBACK_TOKEN`. | Виджет отзыва скрыт; `POST /feedback` → `503`. |  |
