# Инструкция пользователю — загрузка счетов

`Аудитория: сотрудник, который загружает счета.` Подробное руководство (для админа/владельца тоже) — [USER_GUIDE.md](USER_GUIDE.md).

## 1. Где открыть
- **Внутри Битрикс24** — приложение в левом меню портала. Вход не нужен — сессия открывается сама.
- **По отдельной ссылке** (вне портала) — форма **«Вход»**: логин и пароль (выдаёт администратор).

## 2. Что вставить
1. Перетащите файлы в зону **«Перетащите файлы сюда»** (или нажмите и выберите). Можно несколько — **до 10 файлов, по 20 МБ**.
2. Форматы: **PDF, фото (JPG/PNG), Excel (XLSX/XLS), Word (DOCX)**.
   - У Excel читаются **все листы** (реквизиты и товары могут быть на разных).
   - `.xls`, который на деле новый Excel (выгрузка из 1С) — тоже распознаётся.
3. Нажмите **«Загрузить N файлов»**. _(До нажатия лишний файл можно убрать крестиком.)_

## 3. Как получить результат
- Идёт обработка — виден прогресс и таймер ⏱. **Один файл может разбираться до нескольких минут — это норма** (особенно скан/фото — через распознавание).
- Файлы идут **по очереди**. Можно **«Остановить импорт»** (вверху) или **«Убрать»** файл, который ещё в очереди («Ожидание»).
- Вкладку лучше не закрывать.

### Что вносится в сделку (тезисно)
- Воронка **«Закупки»**, валюта **BYN**, НДС **20%**, единица — **штуки**.
- **Поставщик** — ищется по **УНП** из счёта.
- **Договор** — ищется по номеру/дате из счёта (если есть).
- **Позиции** — добавляются, только если **артикул есть в каталоге**.
- **Цена** — вносится **1-в-1** со счётом; «Итого»/«НДС» считаются по счёту.
- **Ответственный** — вы (кто загрузил внутри Битрикс24).

## 4. Результат по каждому файлу
- ✅ **«Сделка #N»** + «Открыть сделку» — готово.
- ⚠️ **«Не внесено позиций: N»** (рядом с ✅) — **не ошибка**: сделка создана, но эти позиции в неё не вошли (нет артикула / артикул не в каталоге). **Заведите их вручную.**
- ⚠️ **«Без сделки»** + причина (напр. «Валюта не BYN», «Иностранный поставщик») — это **бизнес-правило**, не сбой приложения.
- ❌ **«Ошибка»** — файл не удалось обработать (битый/не тот тип).
- **«Лог обработки»** (раскрыть) — что распознал агент и почему такой итог.

## 5. Если что-то не по плану
| Ситуация | Что делать |
|---|---|
| ❌ **«Ошибка»** | Повторите загрузку. Повторяется — **сообщите администратору** (приложите файл/скрин). |
| ⚠️ **«Не внесено позиций: N»** | Откройте сделку и **заведите эти позиции вручную**. Если артикула нет в каталоге — товар нужно завести в каталог. |
| ⚠️ **«Без сделки» — поставщик не найден** | Сделка всё равно создаётся **без компании** (УНП в заголовке) — **привяжите компанию вручную**. |
| ⚠️ **«Без сделки» — иностранный поставщик / не BYN** | Так задумано (отсекаем намеренно). Нужна другая логика — к администратору. |
| Цена/итог в сделке выглядят неверно | Оставьте **👎** с комментарием — к отзыву **приложится файл**, разберём быстро. |
| Долго обрабатывается / зависло | Дайте до ~6 минут на файл. Дольше — остановится с ошибкой, **повторите**. |

## 6. Отзыв (помогите улучшить)
Под каждым файлом: **👍 Хорошо** / **👎 Проблема** (комментарий — по желанию). Особенно полезно **👎 на проблемных счетах** — мы получаем сам файл и быстро воспроизводим случай.
